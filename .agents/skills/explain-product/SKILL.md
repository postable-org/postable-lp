You are working on the Postable landing page.

## What this skill is for
Use this skill whenever the task involves:
- writing or rewriting landing page copy
- creating hero sections, CTAs, FAQ items, feature blocks, or benefit sections
- defining landing page structure and section order
- refining positioning, messaging, and conversion language
- adapting copy to a startup, SaaS, or SMB audience

## Source of truth
Before writing anything, read [product-context.md](product-context.md).
That file contains the canonical product context, audience, pain points, positioning, tone, value proposition, and messaging priorities.

If there is any conflict between assumptions and the context file, follow `product-context.md`.

## Core product understanding
Postable is not just an AI content generator.
Postable is a strategic content and growth platform for SMBs that helps them turn trends into personalized social content designed to generate leads.

The landing page must clearly communicate:
1. who the product is for
2. what painful problem it solves
3. why generic tools and manual posting are not enough
4. how Postable helps generate leads
5. why a visitor should try it now

## Messaging rules
Always keep the message:
- clear
- concrete
- business-oriented
- persuasive without hype
- focused on outcomes

Always emphasize:
- lead generation over vanity metrics
- strategy over generic AI generation
- personalization over templated content
- consistency as a growth driver
- simplicity and time savings for SMBs

Avoid:
- empty buzzwords
- enterprise-style jargon
- generic startup filler
- overexplaining AI as the main value
- making the product sound like "just another post generator"

## Writing rules
When writing copy:
- write for SMB owners and lean marketing teams
- use short, high-clarity sentences
- make pain points easy to recognize immediately
- connect inconsistent posting to lost leads and weak growth
- show that Postable reduces dependence on agencies, freelancers, and manual effort
- make the copy sound strategic, practical, and conversion-aware
- prefer specific business outcomes like more leads, more visibility, stronger brand presence, and better customer flow

## Landing page output standards
When asked to generate a landing page or section:
- start with the clearest possible positioning
- keep the headline outcome-driven
- support each claim with practical value
- include CTA suggestions where useful
- structure copy so it feels ready for a real SaaS landing page

For a full landing page draft, prefer this flow:
1. Hero
2. Trust or credibility statement placeholder
3. Problem
4. Solution
5. How it works
6. Benefits and outcomes
7. Why Postable instead of agencies or manual work
8. Features
9. FAQ
10. Final CTA

## Section-specific guidance
### Hero
The hero should quickly explain:
- the audience
- the main pain
- the core outcome

Good hero copy should make the visitor feel:
- this is for businesses like mine
- this solves a real problem I already have
- this can help me generate more leads with less effort

### Problem section
Make the pain feel real:
- SMBs do not know what to post
- they do not post consistently
- inconsistency weakens lead generation and growth
- manual content work costs time they do not have

### Solution section
Frame Postable as:
- strategic, not generic
- personalized, not one-size-fits-all
- built for lead generation, not just engagement
- simpler and more accessible than hiring outside help

### CTA
CTAs should feel direct and practical.
Examples of direction:
- Start your 7-day free trial
- Generate your first strategic posts
- Stop guessing what to post
- Turn trends into leads

## If the user asks for only one part
If the request is narrow, produce only the requested asset.
Examples:
- hero copy only
- FAQ only
- section ideas only
- headline options only
- copy critique only

## If the user asks for multiple options
Provide clearly differentiated variants.
For example:
- more direct
- more premium
- more startup-like
- more conversion-focused

## Final check before responding
Before finalizing, verify that the output:
- sounds like Postable, not a generic AI product
- speaks to SMB pain clearly
- highlights lead generation
- reflects the context in `product-context.md`
- is easy to place into a real landing page
